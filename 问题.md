# (二)c语言中如何修改指针本身的指向而非指针指向的内容（指针的指针）

以2.2的结构体为例，假如我这里有一个结构体，然后我给这个结构体起别名叫lnode(普通结构体别名)和*linklist(指针型j结构体的别名,其实和lnode * 相同)，在结构体里有两个变量data,next，这就符合了链表的特征（后面就叫他为链表）

~~~c
typedef struct {
    int data;
    int next;
}lnode,*linklist;
~~~

若想修改结构体内容则用

~~~c
int main(void){
    linklist p=(linklist)malloc(sizeof(lnode));//这里修改的只是指针指向的内容，设一个普通的指针型变量p就行
    if(p==NULL)return 0;
    p->data=2;
    return 0;
}
~~~

若想修改链表指向则要用指针的指针

~~~c
int main(void){
    linklist *p=(linklist*)malloc(sizeof(lnode));//为指针的指针即头指针分配内存，linklist本身就是一种指针，现在我又有一个指针型的变量*p，这就是指针的指针，后面malloc的类型与变量p的类型一致，即双重指针
    if(p==NULL)return 0;//判断指针的指针p内存是否申请成功
    *p=(linklist)malloc(sizeof(lnode));//为头结点分配内存，此处的*是解码的意思(可以把他当成一个钥匙孔，有了它你才可以得到指针变量p里的东西，此处里面的东西是头结点的地址)
    if(p==NULL){
        free(p);
        return 0;
    }
    (*p)->next=NULL;
    return 0;
}
~~~

### 3.1头文件里要放哪些东西呢

**1.**要放实现这个功能所需的头文件，而不是在源文件里放头文件为什么:

a.避免在调用这个功能时头文件过于冗长,只需函数引用就行提高工作效率

 b.提升项目的可维护性和兼容性，当我需要修改功能时只需在该功能的头文   件里修改 ，无需在每个调用该功能里的源文件里修改

**2.**要实现数据的基本初始化，如链表的创建与初始化，贪吃蛇身体和头的初始化 为什么：

a:首先他们是实现这个项目的基本，每个功能的实现都离不开他们，这么做只需进行函数引用就能实现初始化

## 4.架构实现

### 4.1.首先链表的初始化

仅仅只是初始化，即在计算机里占用一段内存并且没有具体数据，以货架举例相当于我在管理员那里申请了一个空货架

后面的设结点并申请动态内存和next域置空，相当于我以第一此申请的货架为模板，我有几种货物就申请几个这样的货架，货架和货架之间以绳子（指针）相连，最后一个货架不需要绳子（注意链表的每个结点之间不是连续的空间）

### 4.2.尾插法：

**设一个指针指向尾结点**，再设一个添加结点，尾结点的next域指向添加结点，添加结点的next域指向空，指针再指向添加结点即尾结点

### 4.3.删除结点(关注结点删除之后，如何保证链表的连续性)：

设两个指针，一个指针指向头指针，另一个指向头结点，while循环当指针值不为空时，分别指向下一个next域，当找到符合要求的节点时，如果删后继续遍历数组就要单独存储删除结点的next域，否则不需要代码如下

~~~c
int borrow(linklist* L, borrowlinklist* R,char n[40]) {
	linklist p,j;//设指针p,j
	borrowlinklist r, q;//设结点r，指针q
	p = (*L)->next;//p指向首元结点
	j = *L;//j指向头结点，记录前一个结点的地址
	while (p!= NULL) {
		if (strcmp(p->data.name, n) == 0) {//当p->data.name和n完全相同时
			q = (*R)->next;//指向头节点
			r = (borrowlinklist)malloc(sizeof(borrowlnode));//添加结点
			if (r == NULL)return 0;
			r->next = NULL;
			strcpy(r->data.borrowbookname, p->data.name);
			strcpy(r->data.borrowbookid, p->data.id);
			strcpy(r->data.borrowbookwriter, p->data.writer);
			while (q->next != NULL)q = q->next;//尾插法
			q->next = r ;
			r->next = NULL;//保证链表的连续性
			j->next = p->next;//此处没有单独保存p->next域是因为此处删除后不需要遍历
			free(p);
			return 1;
		}
		else {
			j = p;
			p = p->next;
		}
		return 0;
	}
}

~~~

**注意：一般情况下要单独存储删除结点的next域，能避免大部分内存问题**

### 4.4图书管理

## (六)题

2026.1.5leet合并两个有序链表

~~~c
struct ListNode* mergeTwoLists(struct ListNode* list1, struct ListNode* list2) {
    struct ListNode dummy;
    struct ListNode *mer=&dummy;
    dummy.next=NULL;
    struct ListNode *a=list1;
    struct ListNode *b=list2;
    while(a&&b){
        if(a->val>=b->val){
            mer->next=b;
            b=b->next;
        }
        else {
           mer->next=a;
            a=a->next;
        }
        mer=mer->next;
    }
    mer->next=(a==NULL) ? b:a;
    return dummy.next;
}
~~~

1.哑结点：无需先比较两个已有链表的首元结点的大小，直接作为头结点

2.本题是要合并两个链表形成一个新的链表，所以要设一个新的链表mer

3.每当新设一个结点时都要初始化

4.合并链表，注意当一个链表为空后，要把另一个链表的剩下的部分接上去
