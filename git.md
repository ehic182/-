# Git

## 简介

1.备份：通过分布式版本控制，进行共享版本库，即每台设备都可以是版本库(仓库：用来存储数据)，即使一个版本库(仓库)坏了，也可以用其他设备作为库

2.还原代码

3.协同开发时追溯，监控

## 实操

git add 放入缓存区(类似一个中间量，用来存储即将放入数据库的数据)

git commit 提交存储在缓存区的数据

git status 检查数据状态（1.若为红则未放入缓存区 2.若为绿则已放入缓存区 3.若以上两种情况未出现则已将数据存入数据库 ）

git log查看提交日志

git-log更为强大的查看日志（看起来方便但需提前设置：alias git-log='git log --pretty=oneline --all --graph --abbrev-commit'）**常用**

git branch 查看分支

git branch <u>文件名</u> 创建分支

git branch -d <u>文件名</u> 删除分支，若为大d则为强制删除

git checkout <u>文件名</u> 转换分支git

git checkout -b <u>文件名</u> 创建分支并转换分支（实现多人开发）

touch <u>文件名</u> 创建文件

vi 文件名    编辑文件

git reset --hard <u>文件码</u> 切换版本 

git merge <u>文件名</u> 整合分支（将此处文件名整合到HEAD所指向的分支）

## 远端仓库（他也是一个分支）

### 操作

git remote add origin  <u>远端地址</u>  (其中origin是远端标签)设置远端仓库

git remote (查看远端仓库)

git push origin <u>分支名</u> （推送分支到远端仓库）

git push --set-upstream origin master:master (将远端的master分支和本地绑定，下次git push直接推送到当前所在分支gg远端，不用再写远端仓库明和本地分支名了g)

git clone <u>远端地址链接</u> 

git fetch 拉取(从远端拉取，拉取后由于远端分支更新了，但本地分支未更新，所以要将两者合并要用 git merge)

git pull 抓取（直接拉取并合并）

### 编辑文件内容时如何编辑和退出

1.输入vi <u>文件名</u> 之后会跳转界面

2.按i进入编辑模式，可以编写文件内容
